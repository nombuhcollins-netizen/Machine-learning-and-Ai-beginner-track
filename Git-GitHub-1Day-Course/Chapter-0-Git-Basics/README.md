## Key Concepts Summary

| Concept | Definition |
|---------|-----------|
| **Repository** | A folder containing project history |
| **Commit** | A snapshot of changes with message |
| **Branch** | Independent line of development |
| **Staging Area** | Temporary area for changes before commit |
| **HEAD** | Pointer to current location |
| **Merge** | Combining two branches |
| **Conflict** | Overlapping changes that need resolution |

## Common Questions

**Q: Do I need to memorize all commands?**
A: No! You'll use 10-15 commands regularly. Others you can look up when needed.

**Q: Can I undo a mistake?**
A: Yes! Git is designed to be safe. Most mistakes can be undone.

**Q: Will this work on my operating system?**
A: Yes! Git works on Windows, Mac, and Linux. We provide instructions for all.

**Q: How long will this really take?**
A: 2-2.5 hours if you follow it straight through. Budget time for hands-on practice.

## Tips for Success

1. **Type the commands yourself** - Don't copy-paste (improves learning)
2. **Experiment** - Try variations once you understand the basics
3. **Read error messages** - They're usually helpful
4. **Use `git status` frequently** - It's your best tool
5. **Take breaks** - Don't try to rush through
