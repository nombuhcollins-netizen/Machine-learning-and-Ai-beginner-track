## ğŸ”§ Problem-Solving Strategies

When stuck (and you will be!):

### The 15-Minute Rule

1. **Minutes 0-5:** Read error carefully
   - What exactly is the error message?
   - On which line?
   - What were you trying to do?

2. **Minutes 5-10:** Search your knowledge
   - Have you seen this before?
   - Does a previous example help?
   - Check documentation briefly

3. **Minutes 10-15:** Debug systematically
   - Print intermediate values
   - Use debugger if available
   - Test smaller pieces
   - Check assumptions

**If still stuck after 15 minutes:**
- âœ… Take a break (seriously - 10 min walk)
- âœ… Ask on Stack Overflow (with clear question)
- âœ… Join community chat/Discord
- âœ… Try explaining the problem to someone
- âœ… Read the documentation more deeply

### Debugging Mindset

```python
# âŒ Don't do this:
# Try random things until something works

# âœ… Do this instead:
1. Understand exactly what's failing
2. Form hypothesis about why
3. Test hypothesis with small experiment
4. Fix based on confirmed cause
```

**Use the Scientific Method:**
1. Observation: What's happening? (Error message)
2. Hypothesis: Why is it happening? (Initial guess)
3. Experiment: Test hypothesis (Change one thing)
4. Conclusion: Was I right? (Check result)
5. Iterate: Try next hypothesis
